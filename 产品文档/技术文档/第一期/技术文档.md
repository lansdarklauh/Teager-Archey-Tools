# 射箭助手小程序 - 第一期技术实现文档
版本说明：V1.0（第一期纯前端实现）
开发技术：Uniapp + Vue3
数据存储：小程序 storage 本地存储

## 一、整体实现思路
### 1.1 核心功能模块划分
本次第一期开发包含三大核心功能模块，采用 Uniapp+Vue3 纯前端实现，所有数据均存储在小程序 storage 中，模块划分如下：

- 计分模块：包含普通模式、简易模式、自定义模式三种计分模式，实现射箭分数的录入、保存、编辑等核心功能；
- 统计模块：实现计分数据的展示、排序、筛选、图表统计分析等功能；
- 计时模块：实现射箭训练 / 比赛的多阶段计时、提示音、循环计时等功能。

### 1.2 整体架构设计
采用 Uniapp Vue3 的组件化开发架构，遵循 Vue3 的组合式 API 开发规范，整体架构分为三层：

- 页面层：包含计分页面、统计页面、计时页面、个人中心页面等核心页面，负责页面展示和用户交互；
- 组件层：封装通用组件和业务组件，如计分输入组件、靶面配置组件、图表组件、计时组件等，提高代码复用性；
- 工具层：封装本地存储工具、数据处理工具、时间处理工具等，提供通用的工具方法供页面和组件调用。

**数据流转**：用户在页面进行操作，通过组件处理交互逻辑，调用工具层方法进行数据处理和本地存储，最终将处理后的数据渲染到页面。

### 1.3 核心实现原则
- 纯前端实现：不涉及服务端交互，所有数据均通过小程序 storage 进行本地存储和读取；
- 组件化开发：将通用功能和业务功能封装为独立组件，降低代码耦合度，提高可维护性；
- 数据驱动：遵循 Vue3 数据驱动思想，通过响应式数据控制页面渲染和交互；
- 兼容性：适配小程序各端运行环境，保证在不同设备上的正常显示和使用；
- 性能优化：合理管理小程序 storage 存储，避免数据冗余，优化页面渲染效率。

## 三、各模块详细实现思路
### 3.1 计分模块实现思路
计分模块是核心业务模块，包含普通模式、简易模式、自定义模式三种模式，核心实现思路如下：

1. 模式选择：页面初始化展示模式选择入口，根据用户选择的模式加载对应的配置页面，简易模式仅展示预设选项，普通模式展示完整配置选项，自定义模式展示靶面自定义配置；
2. 配置处理：用户完成配置后，对配置参数进行校验（必填项校验、数值合法性校验），校验通过后生成计分输入页面，根据箭数生成对应的分数输入框；
3. 分数输入：采用特制 9 宫格输入组件（1-9、10、X、M），支持点击输入，输入后自动聚焦下一个输入框，自定义模式超过 11 环时切换为普通数字输入框；
4. 计时联动：若开启报时功能，点击开始计分后启动计时，计时过程中支持分数输入，计时结束给出提示音提醒；
5. 拍照功能：开启拍照功能后，每组保存时提醒用户拍照，支持相册选择和临时拍照，限制最多 3 张，拍照后将图片路径存储到 photoList；
6. 分数计算：每箭输入完成后实时计算本组总分和累计总分，11 分制下将 X 转换为 11 分进行计算，自动统计 X 数、10 环数、脱靶数；
7. 组切换：本组分数保存后生成下一组输入框，支持上一组、下一组切换，支持放弃计分和完成计分；
8. 自定义模式：支持最多 6 个靶面的自定义配置，可调整靶面位置、复制表单，多靶面时按小组织分，大组统计子组总分和累计总分；
9. 预设管理：自定义模式支持保存预设（最多 3 个）、加载预设、删除预设，预设包含完整的配置信息；
10. 数据保存：完成计分后，将完整的计分记录通过 storage 工具保存到本地，生成唯一的 scoreRecordId。

### 3.2 统计模块实现思路
统计模块实现计分数据的管理和分析，核心实现思路如下：

1. 数据读取：页面初始化时通过 storage 工具读取本地所有计分记录，按默认排序条件（时间降序）进行展示；
2. 列表展示：展示计分记录的核心信息，包括总分、创建时间、弓种、距离、靶面、组数 / 箭数等，自定义模式仅展示总分、时间、弓种；
3. 排序功能：支持按时间（升序 / 降序）、按分数（升序 / 降序）排序，点击排序按钮后重新处理数据并渲染列表；
4. 筛选功能：支持按时间范围（单日 / 多日）、分数段、弓种进行筛选，用户设置筛选条件后对数据进行过滤并渲染；
5. 详情展示：点击记录进入详情页面，展示规则详情和分数详情，支持分组展开 / 折叠，自定义模式支持靶面详情折叠；
6. 分数编辑：在详情页面支持对已有分数进行编辑，编辑后重新计算总分、累计分等相关数据，并更新本地存储；
7. 图表统计：仅简易模式和普通模式支持图表功能，包含：
   （1）概述模块：计算并展示平均分、中位数、众数；
   （2）统计模块：通过条形图展示环数分布，通过饼图展示环数占比，通过折线图展示分组分数变化；
8. 数据操作：支持计分记录的删除、分享（前端本地分享图片 / 数据）；
9. 分页处理：当计分记录较多时，实现分页加载，提高页面渲染效率。

### 3.3 计时模块实现思路
计时模块实现多阶段的计时功能，支持训练和比赛场景，核心实现思路如下：

1. 时间配置：支持容错时间、准备时间、正式时间、休息时间的自定义配置，其中正式时间为必填项，循环时休息时间为必填项；
2. 预设配置：提供默认预设（0+10+180 单次、5+5+30+30 循环），支持用户选择预设快速配置；
3. 循环设置：支持开启循环计时，休息时间结束后自动进入准备时间 - 正式时间 - 休息时间的循环，可设置循环次数；
4. 提示音设置：支持选择提示音类型（音效 / 语音），默认每个阶段切换时触发提示音，支持在正式时间中添加自定义提示音关键点；
5. 计时逻辑：
   （1）点击开始后，先进入容错时间（若配置），结束后进入准备时间；
   （2）准备时间结束后进入正式时间，正式时间为核心计时阶段；
   （3）正式时间结束后进入休息时间（若配置），循环模式下休息时间结束后重复上述流程；
6. 计时控制：支持开始、暂停、重置计时，实时展示当前阶段和剩余时间；
7. 页面联动：计分模块开启报时时，计时模块与计分模块联动，计时状态实时同步到计分页面。

## 四、推荐组件库与方法库
说明：基于 Uniapp+Vue3 技术栈，推荐适配小程序端的轻量级组件库和方法库，保证小程序运行性能。

### 4.1 基础 UI 组件库
- uView Plus：适配 Uniapp Vue3 的轻量级 UI 组件库，包含按钮、输入框、选择器、弹窗、导航栏等基础组件，小程序端兼容性好，文档完善，适合快速开发；

推荐优先使用 uView Plus，组件轻量化，适配性好，符合小程序的开发需求，可按需引入减少包体积。

### 4.2 图表组件库
统计模块需要实现条形图、饼图、折线图等图表展示，推荐适配小程序的轻量级图表组件库：

- uCharts：专为 Uniapp 开发的图表组件库，支持小程序端，轻量级，支持条形图、饼图、折线图、柱状图等多种图表类型，配置简单，适配性好；

推荐优先使用 uCharts，专为 Uniapp 打造，集成简单，与项目技术栈契合度高，能满足统计模块的所有图表需求。

### 4.3 核心工具方法库
- VueUse：Vue3 官方推荐的工具库，包含大量实用的组合式 API，如 useStorage、useTimer、useSort 等，可简化本地存储、计时、排序等功能的开发；
- lodash-mini：lodash 的轻量级版本，包含常用的数组、对象处理方法，如数组过滤、排序、对象深拷贝等，简化数据处理逻辑；
- dayjs：轻量级的时间处理库，比 moment.js 体积更小，支持时间格式化、时间范围计算等，用于处理计分记录的时间相关逻辑；
- uni-utils：Uniapp 官方提供的工具库，包含小程序端的常用工具方法，如图片处理、设备信息获取等，适配性好；
- custom-storage：自研本地存储工具，基于小程序 wx.setStorage/wx.getStorage 封装，支持数据的增删改查、批量操作，增加数据校验和容错处理。

### 4.4 特殊功能组件
- 九宫格输入组件：自研或基于基础组件封装 9 宫格分数输入组件，支持 1-9、10、X、M 的快速点击输入，实现自动聚焦下一个输入框的功能；
- 图片选择 / 拍摄组件：使用 Uniapp 原生的 uni.chooseImage/uni.chooseMedia API 封装图片选择和拍摄组件，支持图片预览、删除，限制图片数量和大小；
- 提示音组件：基于小程序 wx.createInnerAudioContext API 封装提示音组件，支持音效播放、暂停、音量调节，预加载常用提示音；
- 滑动选择组件：封装距离、靶面、组数 / 箭数等滑动选择组件，提升用户操作体验；
- 折叠面板组件：封装规则详情、分数详情的折叠面板组件，支持多级折叠，适配自定义模式的多靶面详情展示。

## 五、本地存储实现方案
第一期采用小程序 storage 进行本地存储，核心实现方案如下：

1. 存储结构：按模块划分存储 key，采用命名空间的方式避免 key 冲突，如：
   （1）scoreRecords：存储所有计分记录，为数组类型，包含所有 scoreRecord 对象；
   （2）baseConfig：存储基础配置信息，为对象类型；
   （3）timingPreset：存储计时预设配置，为数组类型；
   （4）customPreset：存储自定义模式预设，为数组类型，最多 3 个；
   （5）userInfo：存储用户基础信息，为对象类型。

2. 存储工具封装：封装统一的 storage 工具类，提供以下方法：
   （1）setItem：设置存储，支持数据校验和序列化；
   （2）getItem：获取存储，支持数据反序列化和容错处理；
   （3）updateItem：更新存储，支持局部更新，避免全量覆盖；
   （4）deleteItem：删除存储，支持单个 key 删除和批量删除；
   （5）clearItem：清空指定命名空间的存储，避免误清空所有存储。

3. 数据持久化：小程序 storage 为持久化存储，除非用户手动清理小程序缓存，否则数据会一直保留，满足用户的本地数据存储需求。

4. 性能优化：
   （1）批量操作：对大量数据的操作采用批量读取和保存，减少 storage 的调用次数；
   （2）数据压缩：对较大的图片路径列表、分数列表进行简单的压缩处理，减少存储体积；
   （3）过期处理：为计分记录添加过期标识，支持用户手动清理过期数据，避免存储冗余。

## 六、核心开发规范
1. 代码规范：遵循 Vue3 组合式 API 开发规范，使用 setup 语法糖，代码按页面、组件、工具进行目录划分，命名采用驼峰命名法；
2. 组件规范：组件按通用组件和业务组件划分，通用组件放在 components/common 目录，业务组件放在 components/business 目录，组件命名采用大驼峰命名法；
3. 数据规范：所有响应式数据使用 ref/reactive 创建，避免直接操作 DOM，数据更新通过数据驱动实现；
4. 存储规范：所有本地存储操作必须通过封装的 storage 工具进行，禁止直接调用 wx.setStorage/wx.getStorage，保证数据操作的统一性；
5. 校验规范：所有用户输入和配置参数必须进行合法性校验，包括必填项校验、数值范围校验、类型校验，避免非法数据进入系统；
6. 注释规范：关键代码、复杂逻辑必须添加注释，组件和方法必须添加文档注释，说明功能、参数、返回值；
7. 性能规范：避免在小程序中使用大量的复杂计算，减少页面重渲染，图片采用懒加载，组件采用按需引入；
8. 兼容规范：适配小程序的不同基础库版本，避免使用高版本 API，对低版本 API 进行兼容处理。
